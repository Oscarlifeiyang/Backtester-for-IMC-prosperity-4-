# IMC Prosperity 4 Backtester

[![PyPI Version](https://img.shields.io/pypi/v/prosperity4btx)](https://pypi.org/project/prosperity4btx/)

This repository contains a backtester for [IMC Prosperity 4](https://prosperity.imc.com/) algorithms. The output it generates closely matches the format of the output generated by the official submission environment.

> **Tip:** Want to visualize your backtest results? See the [Visualizer](#visualizer) section below!

## Usage

The package exposes the `prosperity4btx` CLI. If you are working inside this repository without installing the script entrypoint, you can run the same command via `python -m prosperity4bt`.

The algorithm file must be a Python file that exposes a `Trader` class.

Basic usage:
```sh
# Run all available days from round 0 using the bundled repo data
$ python -m prosperity4bt prosperity4bt/trader.py 0 --data prosperity4bt/resources

# Same command if you installed the package entrypoint
$ prosperity4btx prosperity4bt/trader.py 0 --data prosperity4bt/resources
```

Day arguments:
- `<round>` means all available days in that round
- `<round>-<day>` means one specific day
- negative days must keep the extra minus sign, for example `0--1`

Examples:
```sh
# Backtest all available days from round 0, default backtest folder creation
$ python -m prosperity4bt prosperity4bt/trader.py 0 --data prosperity4bt/resources

# Backtest round 0 day 0 only
$ python -m prosperity4bt prosperity4bt/trader.py 0-0 --data prosperity4bt/resources

# Backtest round 0 day -1 only
$ python -m prosperity4bt prosperity4bt/trader.py 0--1 --data prosperity4bt/resources

# Backtest multiple explicit days
$ python -m prosperity4bt prosperity4bt/trader.py 0--2 0--1 0-0 --data prosperity4bt/resources

# Merge profit and loss across selected days when saving output
$ python -m prosperity4bt prosperity4bt/trader.py 0 --merge-pnl --data prosperity4bt/resources

# Keep original timestamps in the merged output log
$ python -m prosperity4bt prosperity4bt/trader.py 0 --original-timestamps --data prosperity4bt/resources

# Save the output log to a custom file
$ python -m prosperity4bt prosperity4bt/trader.py 0 --out prosperity4bt/logs/round0/backtest1.log --data prosperity4bt/resources

# Skip writing a log file
$ python -m prosperity4bt prosperity4bt/trader.py 0 --no-out --data prosperity4bt/resources

# Print the trader's stdout while the backtest is running
$ python -m prosperity4bt prosperity4bt/trader.py 0 --print --data prosperity4bt/resources

# Disable the progress bar
$ python -m prosperity4bt prosperity4bt/trader.py 0 --no-progress --data prosperity4bt/resources

# Change how orders are matched against market trades
$ python -m prosperity4bt prosperity4bt/trader.py 0 --match-trades worse --data prosperity4bt/resources
```

Useful flags:
- `--out PATH`: save the merged backtest log to a specific file
- `--no-out`: do not save a backtest log
- `--vis`: open the local visualizer after the backtest finishes
- `--data PATH`: read CSV data from a directory instead of packaged resources
- `--merge-pnl`: carry profit and loss across selected days in the saved output
- `--original-timestamps`: preserve original timestamps in the saved output
- `--print`: stream trader stdout to the terminal while still recording it in the log
- `--match-trades {all|worse|none}`: configure market-trade matching behavior

Notes:
- `--out` and `--no-out` are mutually exclusive
- `--vis` requires an output log, so do not combine it with `--no-out`
- `--data` should point to a directory shaped like `prosperity4bt/resources`
- **Remember to clear backtestlog or create new logs before running the backtester**



## Order Matching

Orders placed by `Trader.run` at a given timestamp are matched against the order depths and market trades of that timestamp's state. Order depths take priority, if an order can be filled completely using volume in the relevant order depth, market trades are not considered. If not, the backtester matches your order against the timestamp's market trades. In this case the backtester assumes that for each trade, the buyer and the seller of the trade are willing to trade with you instead at the trade's price and volume. Market trades are matched at the price of your orders, e.g. if you place a sell order for €9 and there is a market trade for €10, the sell order is matched at €9 (even though there is a buyer willing to pay €10, this appears to be consistent with what the official Prosperity environment does).

Matching orders against market trades can be configured through the `--match-trades` option:
- `--match-trades all` (default): match market trades with prices equal to or worse than your quotes.
- `--match-trades worse`: match market trades with prices worse than your quotes, inspired by [team Linear Utility's Prosperity 2 write-up](https://github.com/ericcccsliu/imc-prosperity-2).
- `--match-trades none`: do not match market trades against orders.

Limits are enforced before orders are matched to order depths. If for a product your position would exceed the limit, assuming all your orders would get filled, all your orders for that product get canceled.

## Visualizer

This repository now includes a local visualizer served from `localhost`. It reads a saved backtest log and opens an interactive page with:
- product selection
- timestamp slider
- price and trade chart
- profit and loss chart
- sandbox warnings and trader stdout
- expandable charts with a narrower focus window

Run a backtest and open the visualizer immediately:
```sh
$ python -m prosperity4bt prosperity4bt/trader.py 0 --vis --data prosperity4bt/resources
```

If you want to control the output path:
```sh
$ python -m prosperity4bt prosperity4bt/trader.py 0 --out prosperity4bt/logs/round0/backtest1.log --vis --data prosperity4bt/resources
```

Open the visualizer directly from an existing backtest log:
```sh
$ python3 -m prosperity4bt.open prosperity4bt/logs/round0/backtest1.log
```

You can still call the helper directly from Python if needed:
```py
from prosperity4bt.open import open_visualizer

open_visualizer("prosperity4bt/logs/round0/backtest1.log")
```

The visualizer starts a temporary local HTTP server, opens your browser, serves the page and log data, and then shuts the server down after the frontend finishes loading.

## Data Files

Data for the following rounds is included:
- Round 0: Tutorial round with prices and anonymized trades data on EMERALDS and TOMATOES. This is the introductory round to learn the basics of the algorithm.

**Product Limits for Prosperity 4:**
- EMERALDS: 80
- TOMATOES: 80

Conversions are not supported in this version.

## Environment Variables

During backtests two environment variables are set for the trader to know the round and day it's being backtested on. The environment variable named `PROSPERITY4BT_ROUND` contains the round number and `PROSPERITY4BT_DAY` contains the day number. Note that these environment variables do not exist in the official submission environment, so make sure the code you submit doesn't require them to be defined.

## Development

Follow these steps if you want to make changes to the backtester:
1. Install [uv](https://docs.astral.sh/uv/).
2. Clone (or fork and clone) this repository.
3. Open a terminal in your clone of the repository.
4. Create a venv with `uv venv` and activate it.
5. Run `uv sync`.
6. Any changes you make are now automatically taken into account the next time you run `prosperity4btx` inside the venv.# imc4_five_guys_bt
